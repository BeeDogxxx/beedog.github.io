# 🎬 为什么看不到视频背景？

## 问题原因

**直接双击打开 `index.html` 文件是看不到视频背景的！**

这是因为：
- 直接打开使用的是 `file://` 协议
- 浏览器为了安全，会对 `file://` 协议下的视频自动播放进行限制
- 视频需要通过 HTTP 服务器运行才能正常播放

## ✅ 正确的查看方式

### 方法1：使用启动脚本（推荐）

#### Windows 用户：
双击运行 `start.bat` 文件

#### Mac/Linux 用户：
在终端中运行：
```bash
./start.sh
```

然后访问：**http://localhost:8000**

---

### 方法2：使用 Python 手动启动

在 `beedog-website-package` 目录下打开终端，运行：

```bash
python -m http.server 8000
```

然后访问：**http://localhost:8000**

---

### 方法3：使用 VS Code Live Server 插件

1. 安装 VS Code 的 "Live Server" 插件
2. 右键点击 `index.html`
3. 选择 "Open with Live Server"

---

## 📂 完整操作步骤

### Windows 用户：
1. 打开文件夹：`beedog-website-package`
2. 双击运行：`start.bat`
3. 浏览器会自动打开：http://localhost:8000
4. 你应该能看到篝火视频背景了！

### Mac/Linux 用户：
1. 打开终端
2. 进入文件夹：
   ```bash
   cd beedog-website-package
   ```
3. 运行启动脚本：
   ```bash
   ./start.sh
   ```
4. 浏览器访问：http://localhost:8000

---

## 🔍 验证视频是否正常播放

打开浏览器后：
1. 应该能看到首页有篝火视频作为背景
2. 视频会自动循环播放
3. 文字内容清晰可见（有深色遮罩层）

如果还是看不到，按 F12 打开控制台，查看错误信息。

## ❓ 常见问题

### Q: 为什么要用 HTTP 服务器？
A: 浏览器安全策略限制 `file://` 协议下的视频自动播放，必须通过 HTTP 协议访问。

### Q: 没有安装 Python 怎么办？
A: 下载安装 Python：
- Windows: https://www.python.org/downloads/
- Mac: `brew install python3`
- Linux: `sudo apt-get install python3`

### Q: 可以部署到 GitHub Pages 吗？
A: 可以！GitHub Pages 是 HTTP 服务器，部署后视频会正常播放。

---

## 🚀 部署到 GitHub Pages 后视频会正常显示

当你将网站部署到 GitHub Pages 后，视频会自动正常显示，因为 GitHub Pages 使用的是 HTTP 协议。

---

**记住：不要直接双击 HTML 文件！必须通过 HTTP 服务器运行！**
