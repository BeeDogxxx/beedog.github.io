# 🔄 如何更新 GitHub Pages 网站

你已经部署了一个版本，现在想要更新它？很简单！以下是几种更新方法：

---

## 方法一：使用 Git 命令行（推荐）

### 步骤：

1. **进入你的网站文件夹**
   ```bash
   cd beedog-website-package
   ```

2. **查看当前的修改状态**
   ```bash
   git status
   ```

3. **添加所有修改的文件**
   ```bash
   git add .
   ```

4. **提交修改**
   ```bash
   git commit -m "更新：添加视频背景和读书笔记页面"
   ```

5. **推送到 GitHub**
   ```bash
   git push
   ```

6. **等待 1-5 分钟**
   - GitHub Pages 会自动检测到更新
   - 重新部署你的网站
   - 访问你的网站链接查看更新

---

## 方法二：直接在 GitHub 网页上更新

### 步骤：

1. **访问你的 GitHub 仓库**
   - 例如：`https://github.com/your-username/your-repo`

2. **点击 "Add file" → "Upload files"**

3. **拖拽或选择要更新的文件**
   - 可以单个文件更新
   - 也可以批量上传

4. **在底部填写提交信息**
   - 例如：`更新：优化视频背景`

5. **点击 "Commit changes"**

6. **等待 1-5 分钟**
   - GitHub Pages 会自动部署更新

---

## 方法三：使用 GitHub Desktop（图形界面）

### 步骤：

1. **打开 GitHub Desktop**
   - 如果没有安装，去下载：https://desktop.github.com/

2. **打开你的仓库**
   - 选择 `File` → `Open local repository`

3. **查看修改**
   - 在左侧面板可以看到所有修改的文件

4. **填写提交信息**
   - 在左下角输入提交信息

5. **点击 "Commit to main"**

6. **点击 "Push origin"**
   - 将更新推送到 GitHub

7. **等待 1-5 分钟**
   - GitHub Pages 会自动部署更新

---

## 📋 推荐工作流

### 日常更新流程：

```
1. 修改文件（在本地编辑）
   ↓
2. 测试本地版本（运行 start.bat）
   ↓
3. Git add . （添加修改）
   ↓
4. Git commit -m "描述" （提交）
   ↓
5. Git push （推送）
   ↓
6. 等待 1-5 分钟
   ↓
7. 访问网站查看更新
```

## 🔍 如何检查部署状态

### 方法1：在 GitHub 仓库查看

1. 访问你的仓库
2. 点击 "Actions" 标签
3. 可以看到部署进度
   - ✅ 绿色 = 部署成功
   - ❌ 红色 = 部署失败

### 方法2：在 Pages 设置查看

1. Settings → Pages
2. 查看最新的部署状态
3. 点击访问链接查看更新

## ⚠️ 常见问题

### Q1：推送后多久能看到更新？
**A：** 通常 1-5 分钟，GitHub Pages 会自动部署。

### Q2：如何强制刷新浏览器缓存？
**A：** 按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）

### Q3：更新失败了怎么办？
**A：** 检查：
- Git 命令是否正确
- 文件名是否正确（注意大小写）
- 网络连接是否正常

### Q4：如何回退到之前的版本？
**A：**
1. 在 GitHub 仓库点击 "Commits"
2. 找到之前的版本
3. 点击右侧的 `< >` 按钮
4. 创建新分支或直接恢复

## 💡 提示

### 提交信息规范：
```bash
git commit -m "feat: 添加视频背景"
git commit -m "fix: 修复书籍封面链接"
git commit -m "docs: 更新 README 文档"
git commit -m "style: 优化按钮样式"
```

### 批量更新文件：
```bash
# 添加所有文件
git add .

# 添加特定文件
git add index.html
git add css/style.css

# 添加特定文件夹
git add css/
git add js/
```

### 查看提交历史：
```bash
git log --oneline
```

```bash

git remote -v

git remote set-url origin https://github.com/your-username/your-repo.git
```
